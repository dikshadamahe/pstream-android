Step 2: Read the JSX structure
        Each JSX element maps to a Flutter widget:
        div className="flex"   →  Row()  or  Column()
        div className="grid"   →  GridView()
        img src={...}          →  CachedNetworkImage()
        className="rounded-lg" →  borderRadius: BorderRadius.circular(8)
        className="bg-shade-100" → color: AppColors.shade100
        className="text-white"  →  style: TextStyle(color: Colors.white)
        className="opacity-50"  →  Opacity(opacity: 0.5, child: ...)
        onClick={...}          →  GestureDetector(onTap: ...)

Step 3: Extract animation specifics from CSS / JS
        transition: opacity 200ms  →  AnimatedOpacity(duration: 200ms)
        transform: scale(0.97)    →  AnimatedScale(scale: 0.97)
        requestAnimationFrame     →  AnimationController(vsync: this)

Step 4: Port color tokens → AppColors
        Read themes/default.ts once, create AppColors class, reference everywhere

Step 5: Implement in Flutter, wrap with RepaintBoundary for heavy animations

Complete Source Code Map — What to Read, Where, Why

For every Flutter screen, this table gives the exact source file(s) in xp-technologies-dev/p-stream to open first. Read the file, translate JSX → Flutter. This is the primary design workflow.

src/components/media/MediaCard.tsx
→ lib/widgets/media_card.dart

The foundational card widget. Contains: IntersectionObserver lazy-loading (port as Visibility widget), Flare hover effect (port as GestureDetector + AnimatedContainer scale), skeleton loading state (port as Shimmer), progress bar logic (shouldShowProgress: >5% and <90%), WatchedMediaCard for progress overlay.

src/components/utils/Lightbar.tsx + Lightbar.css
→ lib/widgets/lightbar.dart (V1)

Particle system. Particle class has: x, y, dx (velocity x), dy (velocity y), lifetime, age, opacity (fade-in/out curve). ParticlesCanvas uses requestAnimationFrame at 120Hz (port as AnimationController + CustomPainter). Seasonal overrides: Christmas (Dec 24-26) snowflakes, Halloween (Oct 31) ghosts/pumpkins, April 20 weed leaves, random 10% chance ocean mode (fish + sharks). Port seasonal images as Flutter asset images.

src/components/utils/Flare.tsx
→ Used inside media_card.dart (V1)

Mouse-tracking radial gradient. On mobile, replace with tap-responsive glow: GestureDetector onTapDown/onTapUp → RadialGradient centered on tap position via CustomPainter.

src/components/overlays/detailsModal/components/layout/DetailsModal.tsx + DetailsContent.tsx + DetailsBody.tsx
→ lib/screens/detail_screen.dart

Detail modal is a full-screen overlay on web. On mobile Flutter port: full-screen route (not a modal). DetailsModal fetches: getMediaDetails + getMediaLogo. DetailsContent orchestrates: DetailsInfo (title, year, rating, genres), DetailsBody (play button, quality indicator via Trakt release dates, overview), DetailsRatings (TMDB score + Rotten Tomatoes), EpisodeCarousel for TV, PeopleCarousel for cast, TrailerCarousel for trailers. Port EpisodeCarousel as bottom sheet (DraggableScrollableSheet).

src/pages/parts/player/ScrapingPart.tsx + src/components/player/internals/ScrapeCard.tsx + StatusCircle.tsx
→ lib/screens/scraping_screen.dart + lib/widgets/scrape_source_card.dart

⭐ Most important screen. ScrapeCard renders per-provider status. StatusCircle: grey (waiting) → yellow pulsing ring (pending) → green check (success) → red X (failure). ScrapingPart has auto-scroll to active source. Port StatusCircle as AnimationController with RepeatMode for the pulse. Auto-scroll: use ScrollController.animateTo() when new source becomes active.

src/components/player/atoms/ (all 15+ files)
→ lib/widgets/player_controls.dart

Key atoms: Volume.tsx (volume slider + mute), SubtitleDelayPopout.tsx (±0.1s offset), NextEpisodeButton.tsx (appears at 90% through), VolumeChangedPopout.tsx (HUD toast), BackLink.tsx (back nav), LeftSideControls.tsx (play/pause + skip buttons). Port each atom as its own widget method inside player_controls.dart for clean organization.

src/pages/discover/components/FeaturedCarousel.tsx
→ lib/widgets/featured_carousel.dart (V2)

8-second auto-advance carousel at top of discover page. Shows 10 curated items. Renders media logo (transparent PNG from TMDB) over backdrop. Scrapes IMDb ratings in background. Port auto-advance with Timer.periodic. Use PageView with custom page indicator dots. Fetch TMDB logo from /movie/{id}/images with include_image_language=en,null.

themes/default.ts + themes/list/*.ts (23 themes)
→ lib/config/app_theme.dart

Token hierarchy: raw scale (black.c50 to black.c250, purple.c50 to purple.c900, blue, ash, shade) → semantic tokens (background.main, text.default, etc.) → component tokens (video.scraping.loading, video.context.background). Port default theme as AppColors class. MVP uses only default. V2 adds runtime theme switching by recreating ThemeData from selected theme's tokens.

public/ directory assets
→ assets/ in Flutter project

Contains: thumbnail-placeholder.png (missing poster fallback), public/tomatoes/Certified_Fresh.svg + Fresh.svg + Rotten.svg (Rotten Tomatoes icons for V2), public/lightbar-images/ghost.png + pumpkin.png + santa.png (seasonal particle images). Copy all to Flutter assets/.

🔀 Foldable

Adaptive layout

Breakpoint System Implementation

// lib/config/breakpoints.dart
enum WindowClass { compact, medium, expanded }

WindowClass windowClass(BuildContext ctx) {
  final w = MediaQuery.sizeOf(ctx).width;
  if (w >= 960) return WindowClass.expanded;
  if (w >= 600) return WindowClass.medium;
  return WindowClass.compact;
}

// Adaptive grid columns per window class
int gridCols(BuildContext ctx) => switch (windowClass(ctx)) {
  WindowClass.compact  => 2,
  WindowClass.medium   => 3,
  WindowClass.expanded => 4,
};

// Use this pattern in ALL screens:
class AdaptiveHomeScreen extends StatelessWidget {
  @override
  Widget build(BuildContext ctx) {
    return switch (windowClass(ctx)) {
      WindowClass.compact  => HomePhoneLayout(),
      WindowClass.medium   => HomeTabletLayout(),
      WindowClass.expanded => HomeTVLayout(),
    };
  }
}

Navigation Patterns Per Device

// lib/widgets/adaptive_nav.dart
class AdaptiveNav extends StatelessWidget {
  final Widget child;
  const AdaptiveNav({required this.child});

  @override
  Widget build(BuildContext ctx) {
    final wc = windowClass(ctx);
    if (wc == WindowClass.compact) {
      return Scaffold(
        body: child,
        bottomNavigationBar: PStreamBottomNav(), // phone nav
      );
    }
    return Scaffold(
      body: Row(children: [
        PStreamNavRail(), // tablet/TV: left rail
        Expanded(child: child),
      ]),
    );
  }
}

Android TV — D-pad Focus

📺
TV focus traversal is non-trivial. Wrap every focusable widget (cards, buttons, nav items) with Focus widget. Use FocusTraversalGroup to define traversal order. TV users navigate with D-pad only — there is no mouse or touch. Cards must visually indicate focus with a purple border/glow. Use FocusNode.hasFocus to apply focus decoration. Media cards on TV should be larger (portrait 180×270dp vs phone 130×195dp).

Code Review Process

👀
When to review: (1) After Dev 1 completes any service/player file. (2) After Dev 2 completes any screen. (3) Before any PR merge to develop. (4) Mandatory before tagging any release. Use the Prompt 3 template from Section 03 in Antigravity to run an AI code review.

Review Categories

Architecture

Separation of Concerns

  • Widgets contain zero business logic
  • All HTTP calls in services/ only
  • All Hive access in storage/ only
  • Riverpod providers as the only state access point from UI
  • No context.read() inside build()
  • Immutable models (no mutable state in data classes)
Performance

Flutter Performance

  • const constructor on all leaf widgets
  • ListView.builder / GridView.builder — never children: [...] for lists
  • RepaintBoundary around Lightbar, scrape animations
  • CachedNetworkImage everywhere — never plain Image.network
  • No setState in parent when only child needs to rebuild
  • Shimmer shows during loading — no empty containers
UX / Device

UX & Device Support

  • No hardcoded pixel values — all via MediaQuery or AppSpacing
  • Adaptive layout uses windowClass() — not just screen width checks
  • Touch targets ≥ 44dp on phone (use InkWell with onTap)
  • Error states handled — not just happy path
  • TV Focus widget wraps all interactive elements
  • Haptic feedback on card taps
  • Player exits to landscape correctly
Security

Security & Correctness

  • No TMDB token in source — reads from AppConfig.tmdbReadToken
  • No Oracle IP in source — reads from AppConfig.proxyBaseUrl
  • These constants read from --dart-define at build time
  • Hive reads null-checked before use
  • SSE connection has timeout (60s) + blocking fallback
  • Stream headers always passed to media_kit — never open URL bare

providers-api — server.js

// /home/ubuntu/providers-api/server.js
const express = require('express');
const cors = require('cors');
const { makeProviders, makeStandardFetcher, targets } = require('@p-stream/providers');

const app = express();
const PORT = process.env.PORT || 3001;
const PROXY = 'http://localhost:3000';

app.use(cors());
app.use(express.json());

const providers = makeProviders({
  fetcher: makeStandardFetcher(fetch),
  proxiedFetcher: {
    fetch: async(url, init) => fetch(
      `${PROXY}/proxy/${encodeURIComponent(url)}`,
      { ...init, headers: { ...(init?.headers||{}), 'x-proxy-url': url } }
    )
  },
  target: targets.NATIVE,
});

const buildMedia = ({tmdbId,type,title,year,imdbId,season,episode,seasonTmdbId,episodeTmdbId}) => {
  const m = { type, tmdbId:String(tmdbId), title:String(title), releaseYear:parseInt(year) };
  if (imdbId) m.imdbId = String(imdbId);
  if (type==='show' && season) {
    m.season  = { number:parseInt(season), tmdbId: seasonTmdbId||tmdbId };
    m.episode = { number:parseInt(episode||'1'), tmdbId: episodeTmdbId||'0', title:'Episode '+(episode||'1') };
  }
  return m;
};

app.get('/health', (_,res) => res.json({ok:true}));
app.get('/providers', (_,res) => res.json({ providers: providers.listSources().map(s=>({id:s.id,name:s.name,rank:s.rank})) }));

app.get('/scrape', async(req,res) => {
  if (!req.query.tmdbId||!req.query.type||!req.query.title||!req.query.year)
    return res.status(400).json({error:'Required: tmdbId, type, title, year'});
  try {
    const r = await providers.runAll({media:buildMedia(req.query)});
    r ? res.json(r) : res.status(404).json({error:'No streams found'});
  } catch(e) { res.status(500).json({error:e.message}); }
});

app.get('/scrape/stream', async(req,res) => {
  res.setHeader('Content-Type','text/event-stream');
  res.setHeader('Cache-Control','no-cache');
  res.setHeader('Connection','keep-alive');
  res.flushHeaders();
  const ev = d => res.write(`data: ${JSON.stringify(d)}\n\n`);
  try {
    const r = await providers.runAll({ media:buildMedia(req.query), events:{
      init:e=>ev({type:'init',sourceIds:e.sourceIds}),
      start:id=>ev({type:'start',id}),
      update:e=>ev({type:'update',...e}),
      discoverEmbeds:e=>ev({type:'embeds',...e}),
    }});
    ev({type:'done',result:r||null});
  } catch(e) { ev({type:'error',message:e.message}); }
  res.end();
});

app.listen(PORT, () => console.log(`providers-api :${PORT}`));
